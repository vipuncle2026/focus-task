# Focus Task 系统功能介绍

Focus Task 是一款以四象限任务管理为核心的桌面任务工具，支持本地桌面运行、浏览器 Web 访问和 Docker 后端部署。系统由 Vue 3 前端、Tauri 桌面壳和 FastAPI 后端组成，兼顾个人本地使用、局域网部署和后续多端同步扩展。

## 核心能力

### 四象限任务管理

系统以“重要 / 紧急”四象限为主视图，帮助用户按优先级组织任务：

- 重要且紧急：需要立即处理的任务。
- 重要不紧急：需要规划和长期推进的任务。
- 紧急不重要：可委托或快速处理的任务。
- 不重要不紧急：低价值、可删除或延后的任务。

用户可以在象限中创建任务、移动任务、完成任务、删除任务，并通过右键菜单快速调整任务所在象限。

### 任务详情管理

任务详情面板支持编辑完整任务信息：

- 标题
- 备注
- 象限
- 标签
- 开始日期
- 截止日期
- 重复规则
- 开始时提醒
- 截止时提醒
- 过期后提醒
- 创建时间
- 最近变更时间
- 同步状态

详情页已针对桌面窗口尺寸做紧凑布局，窗口缩小时可以滚动查看完整内容。

### 今日与完成视图

系统提供不同任务视图，便于按使用场景查看任务：

- 四象限视图：全局任务优先级管理，支持象限筛选高亮。
- 今日任务：聚焦当天截止的未完成任务。
- 已完成任务：查看完成记录。
- 报告页面：展示任务统计、柱状图、热度日历与完成概览，支持自定义日期范围筛选。
- 总结页面：周总结与月总结，提供核心指标对比、象限健康度、完成任务列表、高光任务和改进建议。

### 周总结与月总结

总结页面是系统的数据分析核心，支持周期性复盘：

- **模式切换**：周报 / 月报一键切换，数据按所选时间范围动态刷新。
- **周期导航**：上期 / 本期 / 下期翻阅，可回看任意历史周期。
- **核心指标 ×4**：完成数、完成率、活跃天数、逾期数，每个指标附带环比对比（与上期数据对比，显示涨跌百分比）。
- **象限健康度**：四个象限的完成占比进度条 + 健康评估标签（如"危机模式""规划充分""浪费时间"等），底部一句话总评。
- **完成任务列表**：该周期内所有已完成任务清单，按完成时间倒序排列。
- **高光任务**：本周期完成的 Q1/Q2 重点任务 Top 3，按象限和时间排序。
- **改进建议**：基于数据模式自动生成 3-6 条建议，分为警告、提醒、鼓励、中性四个级别。

### 搜索与状态统计

顶部支持任务搜索，并显示当前任务统计：

- 当前可见任务总数
- 已完成任务数量
- 完成进度百分比
- 今日焦点摘要

## 同步与离线能力

### 本地缓存

前端会将任务缓存到本地，即使后端暂时不可用，也可以保留本地任务数据。离线修改会标记为待同步，后续恢复连接后自动同步。

### 增量同步

系统后端提供任务同步接口，支持：

- 本地新增任务推送到服务端
- 本地修改任务推送到服务端
- 软删除任务同步
- 从服务端拉取增量变更
- 使用 `client_id` 保证客户端任务身份稳定
- 同一 `client_id` 在不同用户之间隔离

### 同步冲突处理

当本地有未同步修改，同时服务端存在更新版本时，任务会被标记为同步冲突。用户可以在任务详情页选择：

- 保留本地版本
- 使用服务端版本

右下角同步状态会显示冲突数量，便于及时处理。

## 提醒与通知

### 任务提醒

系统支持三类提醒：

- 开始时提醒：根据任务开始时间触发。
- 截止时提醒：根据任务截止时间触发。
- 过期后提醒：任务超过截止时间后触发。

用户可以在设置页配置提醒提前量，例如准点、提前 5 分钟、提前 10 分钟或提前 30 分钟。

### 通知机制

前端优先使用系统 Web Notification 权限模型发送通知；在桌面环境中，也支持通过 Tauri 命令发送桌面通知 fallback。设置页提供测试通知入口，方便确认系统通知是否可用。

### 设置页

设置页支持：

- 默认开始时间
- 默认截止时间
- 提醒提前量
- 通知检查频率
- 通知权限状态
- 测试通知
- 打开系统通知设置

## 账号与安全

### 用户账号

后端支持用户注册、登录和当前用户信息查询。任务数据按用户隔离，每个用户只能访问自己的任务。

### 密码策略

系统支持新账号密码长度限制，同时保留对历史短密码账号的登录兼容，避免旧账号升级后无法登录。

### Token 存储

桌面端使用 Tauri keychain 存储登录 token 和用户名，浏览器模式下使用 localStorage 作为 fallback。

### 配置项

后端通过环境变量控制关键配置：

- `FOCUS_TASK_DATABASE_URL`
- `FOCUS_TASK_SECRET_KEY`
- `FOCUS_TASK_TOKEN_EXPIRE_MINUTES`
- `FOCUS_TASK_PASSWORD_MIN_LENGTH`
- `FOCUS_TASK_CORS_ORIGINS`

真实密钥和数据库文件不应提交到代码仓库。

## 数据库与迁移

### SQLite 默认存储

系统默认使用 SQLite，适合个人桌面应用和轻量部署。

桌面 App 运行时，数据库会存放在系统 App 数据目录，不会写入 `.app` 包内部。

Docker 部署时，SQLite 数据库会持久化到 Docker volume：

```text
focus-task-data:/data/todo.db
```

### Alembic 迁移

后端已引入 Alembic 管理数据库结构变化。服务启动时会自动执行迁移，兼容旧数据库结构。

已有迁移覆盖：

- 初始用户表和任务表
- 开始时间字段
- 通知开关字段
- 任务同步相关字段兼容

### 本地管理脚本

后端提供用户管理脚本，支持：

- 查看用户列表
- 删除指定用户及其任务
- 清空全部用户和任务

## 桌面 App 能力

### Tauri 桌面端

桌面端基于 Tauri v2 打包，支持 macOS `.app` 直接启动。

已实现：

- 自定义中文菜单
- 透明标题栏
- 顶部窗口拖动
- Keychain 存储登录信息
- 原生通知 fallback
- App 启动时自动拉起内置后端
- App 退出时关闭由自身启动的后端

### 内置后端

桌面 App 内置 FastAPI 后端可执行文件。启动 App 时会自动检查 `127.0.0.1:8765`：

- 如果健康检查通过，则复用已有 Focus Task 后端。
- 如果端口未占用，则启动内置后端。
- 如果端口被其他服务占用，会报出明确错误。

前端请求本地后端时带有启动期重试，减少刚打开 App 时后端尚未就绪导致的连接失败。

## Web 与 Docker 部署

### Docker API 部署

系统支持只部署后端 API，适合让桌面 App 连接远程或局域网后端。

```bash
docker compose up -d --build focus-task-api
```

API 默认地址：

```text
http://服务器IP:8765
```

### Docker Web + API 部署

系统也支持完整 Web 版部署：

```bash
cp .env.docker.example .env
docker compose up -d --build
```

默认访问地址：

```text
http://localhost:8080
```

Web 容器使用 Nginx 托管前端静态资源，并将 `/api` 反向代理到 FastAPI 后端。
